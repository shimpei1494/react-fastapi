# レビュー出力フォーマット

## 構造

```
## レビュー結果: {対象ファイルパス}

### 🔴 重大 / 🟡 軽微 / 🟢 提案

| # | 重要度 | カテゴリ | 行 | 内容 |
|---|--------|----------|----|------|
| 1 | 🔴     | セキュリティ | 42 | 外部入力をそのまま SQL に渡している |
| 2 | 🟡     | 可読性   | 10 | 関数名 `process` が処理内容を表していない |
| 3 | 🟢     | パフォーマンス | 78 | `useMemo` でメモ化を検討できる |

---

#### #1 🔴 セキュリティ（L42）

**問題**
外部入力をそのまま SQL クエリに渡しており、SQLインジェクションのリスクがある。

**該当コード**
```python
query = f"SELECT * FROM users WHERE name = '{user_input}'"
```

**修正案**
```python
result = await session.execute(select(User).where(User.name == user_input))
```

---

#### #2 🟡 可読性（L10）

**問題**
`process` という名前では何を処理するのかわからない。

**修正案**
`validate_user_input` など、処理内容がわかる名前にする。

---

#### #3 🟢 パフォーマンス（L78）

**提案**
毎レンダリングで重い計算が走っている。`useMemo` でメモ化することで不要な再計算を避けられる。

---

### サマリー

- 🔴 重大: 1 件
- 🟡 軽微: 1 件
- 🟢 提案: 1 件
```

---

## 重要度の定義

| 重要度 | 基準 |
|--------|------|
| 🔴 重大 | セキュリティリスク・データ破壊・本番障害につながる可能性があるもの |
| 🟡 軽微 | バグや品質低下につながるが、即時影響はないもの（命名・ロジックの誤りなど） |
| 🟢 提案 | 必須ではないが、パフォーマンスや可読性の改善として検討を推奨するもの |

## カテゴリ一覧

`セキュリティ` / `パフォーマンス` / `可読性` / `命名` / `型安全` / `API設計` / `React/Hooks` / `テスト`

## 記載ルール

- 行番号が特定できない場合は「行」欄を `-` にする
- 修正案はコードで示す（コードが不要な場合は文章のみでよい）
- 🟢 提案は必ず「提案」であることを明記し、強制しない表現を使う
- 同一ファイルへのレビューは1つのブロックにまとめる
